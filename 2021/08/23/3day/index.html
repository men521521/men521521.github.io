Git冲突的解决方式

  1. A在自己的文件里面写了一些代码
  2. A提交到了公共的项目里面
  3. 此时B也写了相同的代码
  4. B也提交了
  5. 此时就会出现下图的问题

当B修改完冲突

如何解决

  1. B：git add .
  2. B git commit -m ‘描述信息’
  3. B:git push origin dev
  4. A:在A分支里面 git pull origin dev 重新上传
  5. B:在B分支里面 git push orgin dev 上传

上传

第一步写完代码我们想要上传代码到项目里自己的分支里面需要三步骤

git add .

git commit -m ‘描述话语’

git push origin zhm 将自己写完的代码放到项目里面自己的分支里

必须要在自己本地的zhm里面

然后我们拉远程的代码push到自己的代码里面先查看一下

git branch -av 查看项目里面所有的分支

然后我们在拉别人的分支

再看我们当前是在自己的 zhm 上还是在 dev 如果不在自己的dev上我们利用指令

git checkout dev 切换到自己的分支上去

然后就可以拉取dev里面我们自己的代码以及别人的代码了

git pull origin dev

然后再切换回自己本地的分支zhm

git checkout zhm

然后把所有集合的代码拉到我们本地的代码里面进行合并

git merge dev

然后再放到我们自己的zhm里面

git push origin zhm

最后查看一下状态如果是以下的状态说明操作成功
